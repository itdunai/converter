# Развёртывание на Beget (статический сайт)

Этот проект работает полностью в браузере и не требует SSR/API на сервере.
После сборки получается статический каталог `out`, который можно разместить на обычном хостинге.

## 1) Подготовка локально

```bash
npm install
npm run build
```

После сборки появится папка `out`.

## 2) Что загружать на хостинг

На Beget нужно публиковать **содержимое папки `out`** как корень сайта.

Варианты:

- загрузить `out` через SFTP/архив в директорию сайта;
- или настроить CI/CD, который выполняет `npm run build` и выкладывает `out`.

## 3) Настройки в панели Beget

- тип сайта: обычный статический;
- Node.js приложение и команда `start` не нужны;
- домен должен указывать на директорию, где лежат файлы из `out`.

## 4) Переменные окружения

Для статического билда важны только клиентские переменные (`NEXT_PUBLIC_*`), например:

```env
NEXT_PUBLIC_MAX_FILES_PER_BATCH=20
```

Важно: значение `NEXT_PUBLIC_*` вшивается в сборку.  
Если меняете переменную, нужно снова выполнить `npm run build` и заново залить `out`.

## 5) Проверка после деплоя

1. Откройте `https://ваш-домен`.
2. Проверьте загрузку файла, запуск обработки и скачивание результата.
3. Если видите старую версию — сделайте hard refresh (`Ctrl+F5`) и/или очистите кэш CDN.

## 6) Частые проблемы

### `Unexpected token '<'` в JS-файле

Причина: вместо JS чанка сервер/кэш отдает HTML.  
Решение:

- очистить кэш CDN/браузера;
- убедиться, что загружены свежие файлы из `out`;
- проверить, что путь `/_next/static/*` отдается как файл, а не переписывается на `index.html`.

### Открывается `Index of`

Причина: корень сайта указывает не на собранные файлы.  
Решение: убедиться, что в корневой директории опубликовано содержимое `out`.
