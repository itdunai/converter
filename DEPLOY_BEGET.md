# Развёртывание на Beget (Node.js)

Этот проект — `Next.js` приложение с серверной обработкой изображений (`sharp`), поэтому нужен именно Node.js-хостинг, а не статический.

## 1) Подготовка проекта

Перед загрузкой убедитесь, что в проекте:

- есть `package.json` со скриптами `build` и `start`;
- есть `.env` переменные для лимитов;
- приложение локально запускается без ошибок.

Локальная проверка:

```bash
npm install
npm run build
npm run start
```

## 2) Создание Node.js приложения в Beget

1. В панели Beget создайте сайт/приложение типа **Node.js**.
2. Выберите версию Node.js (рекомендуется 20+).
3. Привяжите домен к этому приложению.

Важно: если домен привязан как обычный статический сайт, вы увидите `Index of`.

## 3) Загрузка кода

Варианты:

- через Git (рекомендуется),
- через SFTP/архив.

Загрузите проект в директорию Node.js приложения.

## 4) Установка зависимостей и сборка

В терминале Beget (в каталоге приложения):

```bash
npm ci
npm run build
```

Если `npm ci` недоступен, можно использовать:

```bash
npm install
```

## 5) Команда запуска

В настройках приложения укажите стартовую команду:

```bash
npm run start
```

## 6) Переменные окружения

Добавьте в переменные окружения Beget:

```env
MAX_PARALLEL_JOBS=3
MAX_FILES_PER_BATCH=20
NEXT_PUBLIC_MAX_FILES_PER_BATCH=20
```

Если Beget назначает внутренний порт автоматически, добавьте/проверьте:

```env
PORT=<порт_из_Beget>
```

После изменения `NEXT_PUBLIC_*` переменных сделайте пересборку:

```bash
npm run build
```

## 7) Перезапуск и проверка

1. Перезапустите приложение в панели Beget.
2. Откройте:
   - главную: `https://ваш-домен`
   - healthcheck: `https://ваш-домен/api/health`

Ожидаемо для healthcheck:

```json
{ "status": "ok", "timestamp": 1234567890 }
```

## 8) Частые проблемы

### Открывается `Index of`

Причина: домен смотрит на статическую директорию, а не на Node.js процесс.  
Решение: перепривязать домен к Node.js приложению в панели Beget.

### Ошибка 502/504

Причина: приложение не запущено, упало при старте или неверная команда запуска.  
Решение:

- проверить логи приложения;
- убедиться, что команда `npm run start` корректна;
- проверить, что `npm run build` прошёл успешно.

### Изменения на сайте не видны

Причина: не выполнена пересборка после правок.  
Решение:

```bash
npm run build
npm run start
```

или перезапустить процесс через панель, если команда запуска уже настроена.

## 9) Рекомендации для продакшена

- Храните секреты только в переменных окружения.
- Ограничьте размер и количество файлов (уже поддержано в проекте).
- Для высокой нагрузки перенесите очередь из памяти в Redis/BullMQ.
